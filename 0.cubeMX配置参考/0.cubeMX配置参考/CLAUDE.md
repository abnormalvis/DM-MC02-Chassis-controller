## Project Overview

Active firmware for the **DM-MC02 chassis motor controller** based on **STM32H750VBT6** (Cortex-M7, 480 MHz). Goal: receive CRSF RC commands, drive two DC motors with cascade PID (current inner loop + speed outer loop), tune PID parameters live via VOFA over USART2.

**Build system**: Keil MDK-ARM — open `MDK-ARM/STM32H743.uvprojx` in Keil uVision 5.

## Architecture

### Current Inner Loop (`Core/Src/main.c`)

```
ADC1 DMA (Ch19=PA5 motor A, Ch4=PC4 motor B, triggered by TIM1_TRGO)
    ↓  raw [0-65535], centered at 32768 = 0 A
ADC_RawToCurrent_mA():  current_mA = (raw / 65536.0f) * 3300.0f / 0.01f
    ↓
PI controller: Kp=1.20, Ki=35.0, Kd=0.0
  integral clamp ±10000 mA, output clamp ±5000 mA, period 20 ms (50 Hz)
    ↓
PWM compare value  ← TO BE WIRED (not yet connected to TIM output)
```

VOFA sends 8 floats/frame @ 50 Hz: target_mA, feedback_mA, output_mA, error_mA, Kp, Ki, Kd, aux_current_mA.

### VOFA Live Tuning Protocol

USART2 RX (921.6 k, interrupt-driven) accepts ASCII commands:
```
#P1=<float>!   → set Kp
#P2=<float>!   → set Ki
#P3=<float>!   → set Kd
#P4=<float>!   → set target current (mA)
```
On any parameter change the integral accumulator is reset. Callback registered via `VOFA_SetParamCallback()`.

### PWM Motor Output (to implement)

Timer configuration (from CubeMX):

| Timer | Channel | Pin | Motor role |
|-------|---------|-----|-----------|
| TIM1 | CH1 | PE9 | Motor A — forward |
| TIM1 | CH3 | PE13 | Motor A — reverse |
| TIM2 | CH1 | PA0 | Motor B — forward |
| TIM2 | CH2 | PA1 | Motor B — reverse |

Timers are initialized but PWM output is not started. Add `HAL_TIM_PWM_Start(&htim1, TIM_CHANNEL_1)` etc. in init, then write compare register `__HAL_TIM_SET_COMPARE()` to set duty cycle.

**Hardware constraints (non-negotiable):**
- PWM frequency ≤ 20 kHz — verify TIM prescaler/ARR gives ≤ 20 kHz
- Duty cycle 0–99% — cap compare register at `ARR - (ARR/100)` minimum; never write full ARR value
- Only one of FWD/REV non-zero at a time; both = 0 → active brake

## Key Constraints

- **Loop timing**: The 20 ms PID loop is polled in `main()` using `HAL_GetTick()`. Do not insert blocking delays here.
- **ADC centering**: Raw ADC 32768 = 0 A (bidirectional Hall-effect sensor, 1.65 V midpoint = 0 A). Negative current = reverse torque.
- **Integral reset on param change**: When a VOFA parameter update arrives, the PID integral and previous error must be cleared to avoid windup transients.
- **Speed outer loop**: When adding, the outer loop output becomes the current setpoint for the inner loop. Run outer loop at a lower rate (e.g., 10 Hz) than the inner loop (50 Hz).
- **RC channel mapping**: CRSF channels are normalized to [−1, 1]. Chassis mixing: `pwm_left = throttle + steering`, `pwm_right = throttle − steering`, then scale to current target range. Use the `map()` utility from `Motor_ Tuning_PID/Core/Src/main.c` or equivalent.
- **RC safety switch**: Dedicate one CRSF channel as a kill switch (following the pattern in `Motor_ Tuning_PID/application/remote_control.h`: any active-safe = 0 → zero all outputs immediately).
